Ajax是一种用于创建更快更好、交互性更强的Web应用程序的技术。

使用Ajax，用户可以创建接近本地桌面应用的直接、高可用、更丰富、更动态的web用户界面

Ajax的使用需要Node JS和Express的使用，Node JS可以使用Nodemon重启服务，避免每次手动开关。

NodeJS启动服务：命令行执行命令node <filename.js>

NodeJS关闭服务：按下Ctrl+C

Nodemon安装方法：安装完NodeJS后执行命令npm install -g nodemon

使用Nodemon开启服务端：nodemon <filename.js>

若报错，则以管理员运行命令行，输入set-ExecutionPolicy RemoteSigned后选择Y

跨域

同源策略

同源策略(Same-Origin-Policy)最早由网景公司提出，是浏览器的一种安全策略。

同源：协议、域名、端口号 必须完全相同

AJAX默认请求发出和返回的两端需要遵循同源策略

违背同源策略就是跨域。

如：当前网页地址为：http://153.32.56.101:8080，则目标请求资源也必须是http://153.32.56.101:8080才构成同源。

如果目标请求资源为http://153.55.86.97:8080，则为跨域请求。

JSONP

JSON with Padding是一个非官方的跨域解决方案，只支持GET请求

一些标签拥有跨域能力(如img,link,iframe,script)，JSONP利用script标签的跨域能力发送请求。

CORS

Cross-Origin-Resource-Sharing 跨域资源共享，是官方的跨域解决方案。

它不需要在客户端左任何操作，完全在服务端处理，支持POST和GET请求。

跨域资源共享标准新增了一组HTTP首部字段，允许服务器声明那些源站可以通过浏览器访问哪些资源。

CORS通过设置一个响应头告诉浏览器，该请求允许跨域，浏览器收到该响应后会对响应放行。

CORS文档：跨源资源共享（CORS） - HTTP | MDN (mozilla.org)
